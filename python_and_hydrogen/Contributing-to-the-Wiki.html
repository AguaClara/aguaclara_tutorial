I want a new tutorial…¶

…and I’ll write it.¶

  1. git clone https://github.com/AguaClara/aguaclara_tutorial.wiki.git
  2. The repo contains folders for each tutorial section. Decide whether you would like to add to an existing section or make a new section.
    • To add a new section: Make a new folder and Markdown file, both named after the new section. The Markdown file should briefly explain the purpose/contents of the new section.
  1. Your new tutorial’s File-Name.md should properly capitalized with dashes instead of spaces. This File Name will be the displayed page title on the wiki.
  2. Write your tutorial like how you’d write a report:
    • Define any terms that may be new to the reader.
    • Write with brevity, but don’t oversimplify concepts.
  3. To add images, use URL links:
    1. Add your images to the <Your-Section>/Images/ folder, or make a new Images folder if it doesn’t exist yet.
    2. Be sure that the Image-Name.jpeg is properly capitalized with dashes instead of spaces, like the Markdown file.
    3. ![](https://github.com/AguaClara/aguaclara_tutorial/wiki/FILE-PATH/OF-IMAGE.jpeg)
  4. Push your changes to the wiki. In the Pages dropdown in the sidebar, check to see if your tutorial looks good.
  5. Open an issue on the aguaclara_research repo with the title “Add <Tutorial Name&gt; tutorial”, and explain its contents.
    • Label it as addition and medium priority.
  6. Once the tutorial has been reviewed and accepted, it will appear in the wiki’s sidebar.

…but I don’t know how to write it.¶

  1. Open an issue on the aguaclara_research repo with the title “Add <Requested Tutorial Name&gt; tutorial”, and explain what you’d like to see in it.
    • Label it as addition and low priority.
    • If you know someone who would be good at writing this tutorial, assign the issue to them.
  2. Once the tutorial has been written, reviewed, and accepted, it will appear in the wiki’s sidebar.

I want to change…¶

…an existing tutorial.¶

  1. git clone https://github.com/AguaClara/aguaclara_tutorial.wiki.git
  2. git branch change-<tutorial-name> (In the repo)
  3. Make your changes to the tutorial.
  4. Push your changes to the wiki.
  5. Open an issue on the aguaclara_research repo with the title “Change <Tutorial Name&gt; tutorial”, and explain the changes you made.
    • Label it as enhancement and low priority.
    • In the issue, write the exact name of the branch you made.
  6. Once the changes have been reviewed and accepted, they will appear in the wiki tutorial.

…how the wiki is organized.¶

  1. git clone https://github.com/AguaClara/aguaclara_tutorial.wiki.git
  2. git branch reorganize-<yourNetID> (In the repo)
  3. Reorganize the wiki tutorials how you’d like.
  4. Push your changes to the wiki.
  5. Open an issue on the aguaclara_research repo with the title “Reorganize <section name, if applicable&gt;”, and explain the changes you made.
    • Label it as enhancement and low priority.
    • In the issue, write the exact name of the branch you made.
  6. Once the changes have been reviewed and accepted, they will appear in the wiki.
